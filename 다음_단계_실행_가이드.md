## 📋 단계별 실행 계획

### 1단계: 새 GitHub 저장소 생성

1. **GitHub 접속**
   - https://github.com 접속
   - `loveallprince` 계정으로 로그인

2. **새 저장소 생성**
   - 오른쪽 상단의 **"+"** 버튼 클릭
   - **"New repository"** 선택

3. **저장소 정보 입력**
   - **Repository name**: `english-vocab` 또는 `vocab-app`
   - **Description**: "영어 단어장 웹 애플리케이션 (Flask)"
   - **Visibility**: Public 또는 Private 선택
   - **Initialize this repository with**: 체크하지 않기 (빈 저장소)
   - **"Create repository"** 클릭

---

### 2단계: 로컬 파일들을 새 저장소에 푸시

#### 방법 1: 새 원격 저장소 추가 (추천)

```bash
# 현재 디렉토리 확인
cd C:\Users\Admin\projects\game_english

# 새 원격 저장소 추가
git remote add vocab-repo https://github.com/loveallprince/english-vocab.git

# 새 저장소에 푸시
git push vocab-repo main
```

#### 방법 2: GitHub 웹에서 직접 파일 추가

1. **새 저장소 페이지 접속**
   - https://github.com/loveallprince/english-vocab

2. **필수 파일들 추가**
   - `Dockerfile`
   - `requirements.txt`
   - `web_vocab_app.py`
   - `templates/index.html`
   - `static/style.css`
   - `static/script.js`
   - `vocabulary.json`
   - `quiz_stats.json`

---

### 3단계: Railway에서 새 저장소 연결

1. **Railway 대시보드 접속**
   - https://railway.app
   - 로그인

2. **프로젝트 선택 또는 새 프로젝트 생성**
   - 기존 프로젝트 사용 또는 새 프로젝트 생성

3. **새 서비스 추가**
   - **"New"** 또는 **"Add Service"** 클릭
   - **"GitHub Repo"** 선택

4. **저장소 선택**
   - `loveallprince/english-vocab` 선택
   - Branch: `main` 선택
   - **"Deploy"** 클릭

---

### 4단계: Railway 설정 확인

1. **Settings → Source**
   - Repository: `loveallprince/english-vocab` 확인
   - Branch: `main` 확인
   - Root Directory: `.` 확인

2. **Settings → Build & Deploy**
   - Builder: `Dockerfile` 확인
   - Dockerfile Path: `Dockerfile` 확인

---

### 5단계: 배포 확인

1. **Deployments 탭 확인**
   - 배포 상태가 "Building" → "Active"로 변경되는지 확인
   - 빌드 로그에 에러가 없는지 확인

2. **서비스 URL 확인**
   - Settings → Domains
   - "Generate Domain" 클릭하여 도메인 생성
   - 생성된 URL로 접속 테스트
