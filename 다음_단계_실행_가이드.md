## 🚀 지금 바로 해야 할 일

### 1단계: Railway 접속 및 로그인

1. **Railway 웹사이트 접속**
   - https://railway.app 접속

2. **계정 확인**
   - 현재 로그인된 계정이 `kwiss22`인지 확인
   - 만약 다른 계정(`loveallprince` 등)으로 로그인되어 있다면:
     - 우측 상단 프로필 클릭 → "Sign Out"
     - "Sign in with GitHub" 클릭
     - `kwiss22` 계정 선택

---

### 2단계: 새 프로젝트 생성

1. **"New Project" 클릭**
   - Railway 대시보드에서 우측 상단 또는 중앙의 "New Project" 버튼 클릭

2. **"Deploy from GitHub repo" 선택**
   - "GitHub Repo" 옵션 선택
   - 또는 "Deploy from GitHub repo" 선택

3. **저장소 선택**
   - `kwiss22/english-vocab` 저장소 선택
   - Branch: `main` 선택

4. **배포 시작**
   - "Deploy" 또는 "Add Service" 클릭
   - Railway가 자동으로 빌드 시작

---

### 3단계: 배포 확인

1. **Deployments 탭 확인**
   - 프로젝트 페이지에서 "Deployments" 탭 클릭
   - 빌드 로그 확인
   - 상태가 "Active"가 될 때까지 대기 (약 2-5분)

2. **에러 확인**
   - 빌드 로그에 에러가 있는지 확인
   - 에러가 있다면 로그 내용을 확인

---

### 4단계: 도메인 설정

1. **Settings → Domains**
   - 프로젝트 Settings 탭 클릭
   - "Domains" 섹션으로 이동

2. **도메인 생성**
   - "Generate Domain" 클릭
   - 또는 "Custom Domain" 입력 (선택사항)

3. **URL 확인**
   - 생성된 URL 복사
   - 예: `https://english-vocab-production.up.railway.app`

---

### 5단계: 접속 테스트

1. **브라우저에서 접속**
   - 생성된 URL로 접속
   - 영어 단어장 웹앱이 정상적으로 로드되는지 확인

2. **기능 테스트**
   - 단어 추가
   - 퀴즈 기능
   - 통계 확인

---

## ⚠️ 문제 발생 시

### 빌드 에러가 발생하는 경우

1. **로그 확인**
   - Deployments 탭에서 빌드 로그 확인
   - 에러 메시지 확인

2. **일반적인 에러**
   - `Dockerfile not found`: Dockerfile이 저장소에 있는지 확인
   - `PORT error`: Dockerfile의 CMD 명령어 확인
   - `Module not found`: requirements.txt 확인

3. **해결 방법**
   - 에러 메시지를 복사해서 알려주시면 도와드리겠습니다!
