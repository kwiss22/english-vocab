# 저장소 불일치 문제 해결

## 🔍 문제 발견

`loveallprince/loveall` 저장소를 확인한 결과:
- ❌ **영어 단어장 Flask 애플리케이션이 없음**
- ✅ **테니스장 통합 플랫폼 (Next.js) 프로젝트가 있음**
- ❌ **Dockerfile이 없음**
- ❌ **Flask 관련 파일이 없음**

이것이 Railway 배포 실패의 원인입니다!

---

## ✅ 해결 방법

### 방법 1: loveallprince/loveall 저장소에 영어 단어장 추가 (추천)

`loveallprince/loveall` 저장소에 영어 단어장 애플리케이션 파일들을 추가합니다.

#### 필요한 파일들:
- `web_vocab_app.py` (Flask 애플리케이션)
- `Dockerfile` (배포용)
- `requirements.txt` (Python 의존성)
- `templates/` 디렉토리 (HTML 템플릿)
- `static/` 디렉토리 (CSS, JavaScript)
- `vocabulary.json` (단어장 데이터)
- `quiz_stats.json` (통계 데이터)
- `Procfile` (선택사항)
- `railway.json` (선택사항)

---

### 방법 2: 새 저장소 생성

영어 단어장 전용 새 저장소를 만들어서 사용합니다.

1. **새 GitHub 저장소 생성**
   - 예: `loveallprince/english-vocab` 또는 `loveallprince/vocab-app`

2. **로컬 파일들을 새 저장소에 푸시**
   ```bash
   git remote add new-repo https://github.com/loveallprince/english-vocab.git
   git push new-repo main
   ```

3. **Railway에서 새 저장소 연결**
   - Settings → Source
   - 새 저장소 선택

---

### 방법 3: 기존 저장소 사용 (kwiss22/loveall)

`kwiss22/loveall` 저장소에는 영어 단어장이 있습니다.

1. **Railway에서 저장소 변경**
   - Settings → Source
   - Repository를 `kwiss22/loveall`로 변경
   - (권한이 있다면)

---

## 🚀 방법 1 상세 가이드: GitHub에 파일 추가

### 1단계: GitHub 저장소 접속
1. https://github.com/loveallprince/loveall 접속
2. `loveallprince` 계정으로 로그인

### 2단계: 필요한 파일들 추가

#### Dockerfile 추가
1. "Add file" → "Create new file" 클릭
2. 파일명: `Dockerfile`
3. 내용 복사 (로컬 Dockerfile 내용)

#### requirements.txt 추가
1. "Add file" → "Create new file" 클릭
2. 파일명: `requirements.txt`
3. 내용 복사 (로컬 requirements.txt 내용)

#### web_vocab_app.py 추가
1. "Add file" → "Create new file" 클릭
2. 파일명: `web_vocab_app.py`
3. 내용 복사 (로컬 web_vocab_app.py 내용)

#### templates/ 디렉토리 추가
1. "Add file" → "Create new file" 클릭
2. 파일 경로: `templates/index.html`
3. 내용 복사 (로컬 templates/index.html 내용)

#### static/ 디렉토리 추가
1. `static/style.css` 추가
2. `static/script.js` 추가

#### 데이터 파일 추가
1. `vocabulary.json` 추가
2. `quiz_stats.json` 추가

### 3단계: 커밋
각 파일 추가 후 "Commit new file" 클릭
